# Code of Conduct

## Our commitment

sekimori is a small, security-sensitive project. We want its issue tracker,
pull requests, and other project spaces to be welcoming and useful to people
of every background and experience level.

Contributors and maintainers are expected to be respectful, assume good
intent, give technically specific feedback, and accept that the project has
deliberately narrow scope.

## Unacceptable behaviour

Harassment, discrimination, personal attacks, threats, sexualised language or
imagery, publishing someone else's private information, and sustained
disruption of project spaces are not acceptable. Disagreement about design is
welcome; hostility toward people is not.

## Enforcement

The maintainer may remove content, limit participation, reject contributions,
or block accounts when this code is violated. The response should be
proportionate to the behaviour and any safety risk.

## Reporting

Do not put sensitive details into a public issue. Security vulnerabilities use
the process in [SECURITY.md](SECURITY.md), not this policy.

Send conduct reports by private X Direct Message to
[@yktsnd](https://x.com/yktsnd). Do not post the report in a public issue,
reply, or mention. If X does not let you open a Direct Message, open a public
issue only to request another private channel; include no report details.
