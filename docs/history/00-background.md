# 背景

> **凍結済みの歴史文書**(ラウンド記録)。現在の挙動・仕様の一次情報源は [README](../../README.md) と `docs/` 直下の各リファレンスを参照。

いま、AI API や生成 AI を使って試作品を作れる人は急速に増えている。一方で、それを実際にアプリとして公開する段階には、共通してこういう障壁がある:

1. **API キーの壁** — キーはクライアントに置けないので、試作品がどれだけ動いていても公開には必ずバックエンドが要る
2. **課金暴走の壁** — 従量課金 API は、悪意・バグ・想定外のバズのどれでも財布に直結する。上限を「確実に」効かせる仕組みを自作する必要がある
3. **悪用の壁** — 認証のない LLM エンドポイントは即座に「無料プロキシ」として発見・濫用される
4. **定型コードの壁** — ストリーミング、リトライ、エラーマッピング、CORS。毎回同じものを書く
5. **プライバシーの壁** — ユーザー入力をそのままログに残すと、それだけで負債になる
6. **運用の壁** — 誰がいくら使ったか、いつ止めるか、を把握・制御する手段

この「**AI を使える**」と「**AI アプリとして出せる**」の間にある障壁を、小さく実用的な道具で解決する方向が、多くの人に必要とされるはずだ — というのが manabi-repeat から持ち帰る結論であり、次のプロジェクト **sekimori** の出発点になる。
